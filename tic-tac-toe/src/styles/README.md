# 🎨 styles/ — SCSS Design System

> All visual styling lives here. 11 SCSS partials + 1 entry point.
> Compiled by Vite (dart-sass) into `main.css` + `main.css.map`.

## 🔑 Import Order (must not change)

```scss
// main.scss
@import 'variables';   // 1 — tokens (no output)
@import 'mixins';      // 2 — patterns (no output)
@import 'reset';       // 3 — normalize
@import 'base';        // 4 — root defaults
@import 'layout';      // 5 — page structure
@import 'components';  // 6 — shared UI
@import 'board';       // 7 — game board
@import 'score';       // 8 — scoreboard
@import 'modal';       // 9 — result dialog
@import 'animations';  // 10 — keyframes
@import 'responsive';  // 11 — breakpoints (always last)
```

Variables and mixins come first because every downstream partial depends on them.
Responsive always comes last so it overrides everything above it.

## 📄 File Responsibilities

### `_variables.scss` — Design Tokens
Single source of truth for every visual value in the project.
**Never hardcode** a color, size, or timing anywhere else — always use a variable.

| Category | Variables |
|---|---|
| Colors | `$color-bg`, `$color-surface`, `$color-x` (blue), `$color-o` (pink), `$color-win` (green), `$color-draw` (amber) |
| Spacing | `$space-1` → `$space-16` (4px increments) |
| Typography | `$font-sans`, `$font-mono`, `$text-xs` → `$text-4xl`, `$weight-*` |
| Borders | `$radius-sm` → `$radius-full` |
| Shadows | `$shadow-sm/md/lg`, `$shadow-x`, `$shadow-o`, `$shadow-win` |
| Transitions | `$transition-fast/base/slow/bounce` |
| Z-index | `$z-base`, `$z-board`, `$z-overlay`, `$z-modal` |
| Board | `$board-size`, `$cell-size`, `$cell-gap`, mobile equivalents |
| Breakpoints | `$bp-sm` (480px), `$bp-md` (768px), `$bp-lg` (1024px) |

---

### `_mixins.scss` — Reusable Patterns
SCSS functions imported by any partial that needs layout or component logic.

| Mixin | Output |
|---|---|
| `flex-center` | `display:flex; align-items:center; justify-content:center` |
| `flex-between` | flex with space-between |
| `flex-column` | flex column direction |
| `flex-column-center` | flex column + centered |
| `card($bg, $border)` | surface card with background + border |
| `card-glass` | glassmorphism with `backdrop-filter: blur` |
| `btn-base` | button reset + cursor + transition + disabled state |
| `focus-ring($color)` | accessible `:focus-visible` outline |
| `glow($color, $intensity)` | `box-shadow` glow effect |
| `mono-tag` | DM Mono, small caps, muted color |
| `respond-to($bp)` | media query wrapper |
| `visually-hidden` | accessible screen-reader-only pattern |
| `lift-on-hover($amount)` | `translateY` on hover |

---

### `_reset.scss` — Browser Normalization
Removes browser default inconsistencies.
`box-sizing: border-box` on all elements, margin/padding zero, font inheritance, button reset, image `display:block`.

---

### `_base.scss` — Root Defaults
Loads Google Fonts (`DM Sans` + `DM Mono`).
Sets CSS custom properties on `:root` for JS access.
Body background, color, font. Scrollbar styling, `::selection` highlight.
Utility classes: `.hidden` (`display:none !important`), `.sr-only` (visually hidden).

---

### `_layout.scss` — Page Structure
`.app` wrapper with animated grid background (`gridDrift`) and ambient glow blobs (`blobPulse`).
`.header` with shimmer title gradient.
`.setup` panel with mode grid and difficulty grid.
`.game` panel with flex column layout.
`.controls` row for game buttons.

---

### `_components.scss` — Shared UI Components
`.mode-btn` — 1v1 / vs CPU selection cards with top accent line, active state glow.
`.diff-btn` — Easy/Medium/Hard with bottom bar color and active state.
`.btn-start` — Full-width gradient button with shimmer hover effect.
`.btn-control` — Secondary / ghost / danger variants.
`.turn-indicator` — Pill with animated dot, X/O/thinking state variants.

---

### `_board.scss` — Game Board
`.board` — CSS grid `repeat(3, $cell-size)` with gap, card surface, border radius.
`.board__cell` — Individual cells with hover scale, X/O state classes, winner highlight, draw fade, disabled pointer-events.
Cell states: `--x` (blue + `::after` ✕ glyph), `--o` (pink + `::after` ○ glyph), `--winner` (green glow + pulse), `--draw` (fade opacity), `--disabled`.
Hover preview: `data-hover` attribute shows ghost symbol on empty cells.
`.board--locked` — disables all empty cells during CPU turn or game over.

---

### `_score.scss` — Scoreboard
`.scoreboard` — 3-column CSS grid (X / Draws / O).
`.score-card` — Surface card with colored `::before` top bar per player.
Variants: `--x` (blue), `--o` (pink), `--draw` (amber).
`.score-bump` animation triggered when a score increments.

---

### `_modal.scss` — Result Dialog
`.modal` — fixed overlay with backdrop blur.
`.modal__backdrop` — semi-transparent click-to-close layer.
`.modal__card` — glassmorphism card with `cardBounceIn` entry animation.
Top accent gradient (`::before`), icon drop animation, score snapshot grid.
`.btn-modal` variants: primary (blue gradient), secondary (surface), ghost (transparent).

---

### `_animations.scss` — All Keyframes
Every `@keyframes` definition lives here — no keyframes scattered in other files.

| Keyframe | Used By |
|---|---|
| `fadeDown` | header entrance |
| `fadeUp` | setup/game panel entrance |
| `modalIn` | modal overlay fade |
| `cardBounceIn` | modal card entrance |
| `iconDrop` | modal emoji |
| `cellReveal` | X/O placed on board |
| `winnerPulse` | winning cells |
| `winnerGlow` | winning cells continuous glow |
| `drawFade` | draw state fade |
| `scoreBump` | score number on win |
| `dotPulse` | turn indicator dot |
| `dotBlink` | CPU thinking dot |
| `gridDrift` | background grid movement |
| `blobPulse` | background glow blobs |
| `shimmer` | start button hover |

---

### `_responsive.scss` — Breakpoints
Mobile overrides using the `respond-to($bp)` mixin.
≤480px: smaller board cells (`$cell-mobile`), reduced padding, adjusted typography, stacked controls, compact modal.
≤768px: adjusted header title size.

## 🔍 Compiled Output

| File | Description |
|---|---|
| `main.css` | Production-ready compiled CSS. Do not edit — auto-generated by Vite/Sass. |

> ⚠️ Never edit `main.css` directly. All changes go in the SCSS partials.
